關鍵差異對照

v1 是「side padding 統一 16/24/32」；v2 是「side padding 全 0，貼齊邊緣」

元素 v1（之前） v2（這次）
Page wrapper side padding16/24/320
每個 block wrapper 寬度viewport - 32/48/64= viewport（觸邊）
每個 block 內部 padding14/18/2214/18/22（不變）
card border-radius12維持 inputs/buttons 8 / pill 999；外層 block 因觸邊→ 不需 radius（square block 反而 OK）
字型system-uisystem-ui（含 grade letter，無例外）
錯誤狀態紅框 + 淺紅底紅框 + 淺紅底（不變）

注意點

  • radius 矛盾：block 觸邊代表 4 個角觸到 viewport 邊，這時候 border-radius 沒視覺效果（只在內部 input/btn 上有意義）。
  • desktop 1280 form 欄位太寬：用 2 欄 grid 解（已示範），1 欄會出現很長一條 input 不好用。
  • NSM home 的 question card 也適用：每張題目卡也觸邊一條，不再是「卡片之間有 16px 縫」。
  • 仍有少數例外：modal / offcanvas / login card 這類「飄起來的卡片」維持有 radius、有 margin（不貼邊）。
