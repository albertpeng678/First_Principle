Loading 細節

Checklist 項目隨進度變化（純視覺，不需要 API 提供）：
畫面 mount 後 0s — 4 顆都 pending；500ms 第 1 顆勾；1000ms 第 2 顆勾；1500ms 第 3 顆轉 spinner；3000ms 第 3 顆勾、第 4 顆轉 spinner。即使 API 還沒回，也能讓使用者「感覺有進展」。
逾時策略：15 秒沒回應 → 顯示 toast「比預期慢一些，再等等…」；30 秒仍無回應 → 進 error state。

Error 細節

錯誤分類與行為：
  • EVAL_TIMEOUT（30 秒未回）→ 顯示「重新評分」+「返回修改答案」
  • EVAL_API_ERROR（OpenAI 5xx 或 4xx）→ 顯示重試 + 返回
  • EVAL_PARSE_ERROR（OpenAI 回的 JSON 格式錯）→ 顯示重試（背後自動重試 3 次後仍失敗才到 UI）
  • EVAL_AUTH_ERROR（401 — token 失效）→ 跳到登入頁，不顯示錯誤頁
「重新評分」：直接重打 evaluate-step API（自動帶上同樣的 framework_draft）
「返回修改答案」：navigate 回 Phase 1，circlesPhase = 1，保留 framework_draft（已存）
「auto-save」保證：framework_draft 在點送出時就 PUT 進 session.step_drafts，不會因評分失敗而丟失

確認

① loading：spinner + checklist（4 顆視覺進度），15 秒慢警告 / 30 秒進 error
② error：3 種錯誤代碼，2 個按鈕（重新評分 / 返回修改），dev 模式或 console 顯示細節 code
