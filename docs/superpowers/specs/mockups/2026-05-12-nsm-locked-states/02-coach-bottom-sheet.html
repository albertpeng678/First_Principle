NSM Step 4 對比 Tab — 教練思路 Bottom-Sheet（Mobile only）

問題：production 目前 mobile 教練思路 overlay 使用 .modal-card（貼 viewport 邊 / 無 handle / 無圓角頂部 / 無 safe-area padding），視覺不專業。
本 mockup 設計新的 bottom-sheet pattern：16px 圓角頂部 + 36×4 handle pill + safe-area padding + backdrop 50% 暗色 + 可滾 body + 了解了 footer btn。
Tablet / Desktop 不變：仍使用既有 inline grid + .nsm-coach-detail 展開 panel。

三欄說明： Mobile 360 — bottom-sheet overlay 啟動狀態（觸及廣度教練版 active，sheet 從底部升起，backdrop 可見）。 Tablet 768 — inline grid layout，無 bottom-sheet（tablet 點擊教練版顯示 inline .nsm-coach-detail panel，不用 overlay）。 Desktop 1280 — 同 tablet，inline grid，無 bottom-sheet。

生產注意：mockup 內 .coach-sheet-overlay 用 position:absolute（因框在 vp-frame 內）； production 實作用 position:fixed + padding-bottom: max(var(--s-4), env(safe-area-inset-bottom))。 Handle 為視覺 affordance，v1 不要求實作 swipe-to-close（click backdrop / 了解了 btn 關閉即可）。

教練思路
觸及廣度應該量到「真正接觸到核心功能（升級提示）」的用戶，而不是平台總曝光人數。 如果用「所有月活」，會把從未看到推廣的用戶也算入，導致觸及指標失真。 精確的觸及定義能幫助團隊找到真實的轉化瓶頸。
為什麼這樣拆解
「觸及」是漏斗最上層 — 它要回答「有多少潛在用戶看到了這個機會」。 建議搭配 instrumentation 確保提示曝光被獨立 log，避免與其他 banner 混淆。
