實作規範（Q3 snapshot guard 是 SP1.5 最關鍵 fix）

  • 偵測時機：Session 還原時（offcanvas click 或 sessionId URL 還原），server 回傳 session record 後，client 端立刻 diff session.question_json.problem_statement vs QUESTION_BY_ID[question_id].problem_statement（normalize whitespace）。如果不同，AppState.circlesStale = true。
  • 顯示永遠走 snapshot：所有題目 UI（題目卡、商業背景、用戶輪廓、教練示範等）一律從 AppState.circlesSelectedQuestion（= session.question_json）讀，禁止用 QUESTION_BY_ID[id] fresh lookup。
  • 後端 server-side AI：gate / evaluate / chat / coach / hint / final-report — 全部從 session record 拉 question_json 當 prompt context（已是 contract，但需要 jest test 覆蓋）。
  • UI 鎖定：AppState.circlesStale === true 時，phase 1 form readonly、phase 2 input disabled、bottom action bar 簡化為「回首頁」。phase 3 評分頁正常顯示（read-only）。
  • Banner：.stale-banner 出現在 .circles-nav 之後。--c-warn-bg 底色、ph-warning-octagon icon。
  • 「查看新版題目」：點下去 navigate('circles') 到題目選擇頁，並 ?question=<id> 預選新版題（不影響舊 session 紀錄）。
  • Audit script：新增 scripts/audit-stale-sessions.js — 掃 DB 列出所有 question_json 與當前 DB drift 的 session id（給營運觀察用，不刪資料）。
  • Test：jest 測 stale 偵測 + snapshot 顯示（mock session record with old snapshot vs current DB），Playwright 跑 stale session 流程。
