  • B2 上一步 button：在 renderCirclesPhase2() 的 .circles-submit-bar 區塊（line ~3782 結尾）加入 <button class="circles-btn-secondary" id="circles-p2-prev-phase"><i class="ph ph-arrow-left"></i> 上一步</button>，所有 step 都顯示。
  • 點擊 handler：circles-p2-prev-phase click → AppState.circlesPhase = 1; navigate('circles')。framework_draft 已存在 AppState，phase 1 自動 pre-fill。
  • 已得分判定：var isLocked = !!(AppState.circlesStepScores && AppState.circlesStepScores[stepKey])。phase 1 / phase 2 都讀這個 flag。
  • 已得分 phase 1 read-only：buildFieldGroupHtml() 加入 readonly 屬性 + .locked class（CSS 補新規則 background: var(--c-locked-bg); color: var(--c-text-2); cursor: not-allowed;）。submit-bar primary 改為「下一步」（next step or 回評分）。
  • 已得分 phase 2 read-only：.circles-input 加 disabled，.circles-send-btn 加 disabled。submit-bar 兩個按鈕：「上一步（看框架）」 + 「回評分」。
  • locked-banner：在 .circles-nav 之後渲染，使用 --c-success-soft 底色 + --c-success 文字 + score pill (success bg)。
  • conversation 保留：phase 1 編輯送出 → re-run gate API → 成功後回 phase 2，AppState.circlesConversation 不清空。
