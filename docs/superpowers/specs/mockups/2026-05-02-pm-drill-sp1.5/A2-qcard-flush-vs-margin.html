SP1.5 — q-card flush (a) vs margin (b)

A2 navbar + step header 已 edge-to-edge。問題：q-card 要不要也貼邊？兩個方案並排比較。紅斜紋條 = 留白範圍。

方案 (a) — q-card flush 邊到邊 推薦

與 SP1 「貼齊兩側」一致；border-radius 0 + 左右 border 移除（變成 sticky banner 風）；內部 padding 14px 16px 提供 text gutter。

Pros (a) flush

  • 跟你 standing rule「貼齊兩側」一致
  • 視覺極簡，內容區佔滿，沒有 inconsistency
  • 實作 1 行 CSS：.circles-q-card { margin:0 0 12px; border-left:none; border-right:none; border-radius:0 }

Cons (a) flush

  • 左右 border 變透明、border-radius 失效，「卡片感」減弱
  • 需要逐個檢查所有引用 q-card 的地方（home picker / phase 1 / phase 2 / score）是否都適用 flush
  • desktop 寬螢幕下會非常寬（除非 phase1-desktop 自帶 max-width 居中）

方案 (b) — q-card 維持 16px gutter 現狀

navbar 已邊到邊；只 q-card 保留 16px 左右 margin，做為「呼吸感」設計選擇。紅斜紋條 = 16px gutter（不是 bug）。

Pros (b) margin

  • 卡片感清晰，有圓角、四邊都有 border
  • 傳統行動版設計慣例，視覺呼吸感舒服
  • desktop 寬螢幕也不會「太寬」（自然就有外緣留白）

Cons (b) margin

  • 跟你 standing rule「貼齊兩側」不一致 — 對話過程中已點過 4 次
  • navbar / 字母圈 / .circles-nav 已經邊到邊，q-card 自己留 gutter，視覺斷層

我的判讀

(a) flush 跟你 standing rule 一致；(b) margin 是現狀。一致性 > 卡片美感 ⇒ 推 (a)。但 (b) 也合理，看你的權衡。
