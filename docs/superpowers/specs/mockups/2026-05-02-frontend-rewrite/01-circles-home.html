商業背景

Spotify · Spotify Podcast — 練 R · 發掘需求

個別 R·Spotify·Spotify Podcast·難度 中

完整題目

既有用戶聽 Podcast 的留存比聽音樂低（30 天留存 35% vs 音樂 62%）；產品經理需要找出主因並提出策略，但不能影響廣告收入或訂閱轉換率。本次練習聚焦於 R 步驟：挖掘用戶真正的痛點，區分功能性 / 情感性 / 社交性需求，並指出核心痛點。

深入分析

商業背景
Spotify 廣告收入仰賴 ad-supported tier；Podcast 與音樂共享聽眾池但商業模型不同。Podcast 留存低 → 壓縮整體 ARPU + 降低 ad inventory，所以解法不能單純犧牲廣告體驗。
用戶輪廓
免費 + 訂閱混合的通勤 / 運動 / 開車場景用戶。Podcast「subscribe → listen」漏斗轉換差，多數人訂閱後 7 天內未開啟單集 — 顯示 onboarding 與發現機制有 gap。
常見誤區
只列「想要更好的推薦演算法」這種空話 — R 步驟要分到功能 / 情感 / 社交三層。例如「不想被打斷情緒」是情感層，「想跟朋友討論最新一集」是社交層；單壓功能層會錯過真正的 retention drivers。
破題切入
用戶真正的痛點不是「找不到好聽的」，是「沒有時間 / 心境去聽長內容」 — 這是情感層的核心痛點。Podcast 比音樂需要更高的注意力承諾，這個摩擦是基礎假設，所有方案都要繞著它設計。
