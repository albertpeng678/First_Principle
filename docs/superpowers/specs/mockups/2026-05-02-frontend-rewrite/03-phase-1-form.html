iOS Sticky bar 行為（master-spec §0.2 #8）

  • iOS Safari 鍵盤彈出時：用 interactive-widget=resizes-visual viewport meta 讓 visualViewport 縮 — sticky bar 自動維持「鍵盤上方」

  • iPhone notch：viewport-fit=cover + safe-area-inset-top 讓 navbar 不被 notch 蓋
  • 橫向滾動防止：sticky bar position: sticky; bottom: 0 而非 fixed — 配合 visualViewport API 自動跟鍵盤
  • Save indicator 鏡像：floating top-right + sticky bar 內 同時顯示同一狀態 — 用戶從輸入區往下提交不需「找」儲存狀態

如何提升 Spotify Podcast 的用戶留存率？

既有用戶聽 Podcast 的留存比聽音樂低（30 天留存 35% vs 音樂 62%）；產品經理需要找出主因並提出策略，但不能影響廣告收入或訂閱轉換率。請設計改善方案，包含目標族群定義、優先解的痛點、提出 2-3 個方案並評估取捨。

深入分析

商業背景
Spotify 廣告收入仰賴 ad-supported tier；Podcast 與音樂共享聽眾池但商業模型不同（廣告 vs 訂閱）。Podcast 留存低會壓縮整體 ARPU + 降低 ad inventory，所以解法不能單純犧牲廣告體驗。
用戶輪廓
免費 + 訂閱混合的通勤 / 運動 / 開車場景用戶。Podcast「subscribe → listen」漏斗轉換差，多數人訂閱後 7 天內未開啟單集 — 顯示 onboarding 與發現機制有 gap。
常見誤區
把音樂留存策略直接套用到 Podcast — 但兩者的「習慣養成路徑」不同：音樂是 mood-driven、隨手即用；Podcast 是 content-driven、需要更高的注意力承諾。
破題切入
先界定「留存」具體指什麼：訂閱留存（subscribe 後 N 天回來播放）、播放留存（單集完播率）、還是回流留存（30 天再次開啟）？這三種的解法路徑完全不同，所有後續分析都依賴這層界定。
