  • Dismiss：5s 後 fade-out OR 點 ph-x 主動關
  • 沒 session 時：不顯示 banner（避免無謂提示）

Token 逾期 — 重新登入提示

登入逾期，請重新登入
為保護帳號安全，登入狀態定期失效。你的草稿已存於本機，重新登入後會接續到剛才的位置。

技術細節

  • 觸發：API 回 401（Bearer token expired / invalid）
  • 非阻擋式：warn-tinted card 浮在當前畫面頂端（不蓋住所有內容）— 用戶可以先把目前輸入存起來
  • preserve view：點重新登入 → 紀錄當前 `AppState.view + circlesPhase` → login → 登入成功後 redirect 回原處
  • 草稿保留：Phase 1 framework_draft 已 auto-save 到 localStorage（master spec §1.1 PATCH /progress）— 重新登入後從 server 拉同步
  • Auth context loss：navbar 暫時還顯示 email + sign-out，因為前端 state 還沒清；點重新登入後才清

E · Auth flow 整合：

  • 已登入 navbar：email truncate（max 180px）+ ph-sign-out icon button；mobile 寬度不足時 email 隱藏
  • 登出：立即執行 — 清 Supabase session + localStorage token + AppState reset → guest mode；不彈確認彈窗（UX 簡單，反正可以再登入）
  • Migration：guest sessions ≥ 1 + 第一次登入成功 → 自動 POST /api/migrate-guest → 落地頁顯示 success-lt 「已合併 N 場」banner（5s auto-dismiss）
  • Token expiry：API 401 → warn-tinted card 浮在內容頂；preserve 當前 view 到 localStorage → 登入後 redirect 回；草稿不會丟（已 auto-save）
不做的：無確認彈窗的 logout 流（UX 簡）/ 無 email 驗證信（production 已 lock email_confirm: true）/ 無 magic link / 無 OAuth。
Production 來源：app.js:5029 `renderAuth` / Supabase JWT Bearer / X-Guest-ID UUIDv4 / `/api/migrate-guest` (master-spec §1.1)
