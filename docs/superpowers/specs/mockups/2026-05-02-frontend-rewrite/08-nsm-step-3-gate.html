NSM 審核 Gate — 5 criteria 三態 + Loading（紅 = 必擋 · 無 simulation override）

NSM 審核 Gate：AI 對學員填寫的 NSM 定義跑 5 維度檢核： 價值關聯性 / 領先指標性 / 操作性 / 可理解性 / 週期敏感（見 nsm_trainer_full_spec.md）。 三態 + Loading：A 全綠 ok / B 含黃 warn（仍通過）/ C 含紅 error（必擋）/ D Loading 審核中。 紅 = 必擋規則：任一紅 item =「返回修改」唯一動作，無「帶風險繼續」、無 simulation override，drill 與 simulation 行為一致（Master Spec § 2.4）。 Component CSS 整段 LOCKED copy 自 04（gate-transition / gate-item / gate-loading）+ 03（navbar / phase-head / submit-bar / btn）。

5 維度檢核4 OK · 1 WARN
價值關聯性
直接對應 GMV
成交且不退款的下單數和 take rate 直接相乘＝營收，是純度很高的價值代理。
領先指標性
能預測月營收
月初下單上升通常領先當月 take rate 收入 7-10 天，可作為調節廣告投放的訊號。
操作性
部分依賴外部因素
「成功支付」受第三方金流穩定度影響，PM／工程不一定能完全控制；遇到金流故障 NSM 會誤判團隊績效下滑。
建議： 補一個次指標「下單成功率」（成功 ÷ 嘗試）監控金流健康，當 NSM 異常下跌先看次指標排除外部干擾。
可理解性
跨部門共識高
營運／財務／工程都用同一個語意，不需要再翻譯成商業語言。
週期敏感
月週期適中
電商促檔多以月為單位，月活週期可承擔大促波動又能對齊財務報表。

5 維度檢核2 ERROR · 1 WARN · 2 OK
價值關聯性
這是經典的虛榮指標
DAU 翻倍不一定代表 Spotify 賺更多錢——大量「打開 app 但沒聽完一首」的用戶會被算進去，但對訂閱／廣告營收的拉動極弱。
修正方向： 改成「行為深度」型指標，例如「每月完成至少一首完整曲目播放的活躍用戶數」，這樣翻倍才真的對應價值。
領先指標性
無法預測營收走向
DAU 與訂閱續費 / 廣告 CPM 的相關性低；歷史上 DAU 持平甚至上升的月份，營收仍可能下滑（活躍但不深度的觀眾不會付費也不耐廣告）。
修正方向： 先想 Spotify 的 AHA 時刻是什麼動作（例：聽完整首歌 / 收藏第一首）— 那個動作的次數才是真正的領先指標。
操作性
過度仰賴行銷推廣
DAU 容易被獲客成本（廣告投放、促銷）短期拉抬，但這不代表產品變好。
可理解性
直觀易懂
DAU 是業界共通語言，溝通成本低（這是它唯一的優點）。
週期敏感
每日可觀測
DAU 即時可看，A/B 結果回饋很快。
