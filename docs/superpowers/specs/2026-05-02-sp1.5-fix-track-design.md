## 1 · Goal

Close the visual / functional / data-integrity gaps surfaced after SP1 ship, ahead of SP2:

| Track | Item | Severity |
|---|---|---|
| **A1** | Header icon buttons (`.btn-icon` / `.btn-home-icon`) lack border-radius | medium · visual |
| **A2** | Mobile (`#app: 0 16px`) and tablet (`#app: 0 24px`) page wrapper still has horizontal padding → navbar bg has 8px gap with sharp corner; q-card not edge-to-edge | medium · visual / consistency |
| **A3** | Phase 2 renders **two** question cards (`renderPersistentQuestionChip()` + `pinnedCard`), the lower one duplicates statement and creates dead vertical space | low · visual |
| **B1** | A scored step (`circlesStepScores[stepKey]` exists) is still re-editable from "返回" — risks lost evaluation context | medium · UX safety |
| **B2** | Phase 2 has no "上一步" button; user cannot return to phase 1 of same step to review framework input | high · core UX |
| **C1** | 100/100 questions in `circles_plan/circles_database.json` lack `analysis.business / users / insight` → frontend permanently shows fallback "分析資料載入失敗" | medium · data |
| **Q3** | Sessions created before 2026-04-29 question regeneration: `question_json` snapshot drifts from current DB. UI re-fetches by id → wrong question shown for old session, scoring mis-attributed | **CRITICAL · correctness** |

## 4 · Architecture summary

### 4.1 Visual fixes (A1 / A2 / A3) — CSS + targeted JS deletions

- A1: extend `.btn-icon` rule with `width: 40px; height: 40px; padding: 0; border-radius: 50%; justify-content: center` and hover `background: rgba(0,0,0,0.06)`. `.circles-nav-back` already 50%, untouched.
- A2: 
  - `#app` mobile padding `0 16px 16px` → `0 0 16px`
  - `#app` tablet `0 24px 24px` → `0 0 24px`
  - `.navbar` `margin: 0 -16px` removed (no longer needed)
  - `.circles-q-card` (mobile/tablet only) `margin: 0` + `border-radius: 0` + `border-left/right: none`. Desktop keeps centered card with rounded corners (via `.phase1-desktop` 94vw constraint).
- A3: delete `pinnedCard` variable and its render reference in `public/app.js`; delete `circles-pinned-toggle` click handler; delete `.circles-pinned-card` rule from `public/style.css`.

### 4.2 Already-graded read-only (B1) + phase 2 back button (B2) — JS + CSS

- New AppState helper: `isStepLocked(stepKey)` returns `!!(AppState.circlesStepScores && AppState.circlesStepScores[stepKey])`.
- Phase 1 render path: when locked, every `<textarea class="circles-field-input">` becomes `readonly` with class `locked`. Submit-bar shows 「回評分」 (secondary) + 「下一步」 (primary).
- Phase 2 render path: when locked, `<textarea class="circles-input">` becomes `disabled`, `.circles-send-btn` becomes `disabled`. Submit-bar shows 「上一步（看框架）」 + 「回評分」.
- Phase 2 (unlocked): submit-bar gains a new 「上一步」 button (secondary) that sets `AppState.circlesPhase = 1; navigate('circles')` and preserves `AppState.circlesConversation` and `AppState.circlesFrameworkDraft`.
- Phase 1 re-submit (when conversation already exists): `AppState.circlesConversation` is preserved through gate API call; phase 2 re-entry shows previous transcript.

### 4.3 Analysis backfill (C1) — script + data + boundary check

- `scripts/backfill-circles-analysis.js`: change source/target from `public/circles-db.js` to `circles_plan/circles_database.json`. Idempotent (skip questions where `q.analysis.business && q.analysis.users && q.analysis.insight` all populated).
- Run once with OpenAI gpt-4o-mini (~$1) to fill all 100 questions.
- Delete `public/circles-db.js` if it exists (avoid two-source confusion).
- Add jest assertion: every question in JSON has complete analysis post-backfill.
- Add Playwright assertion: home picker sweep does not surface "分析資料載入失敗" string anywhere.

### 4.4 Snapshot guard + stale detection (Q3) — frontend + backend + audit

- Frontend: on session restore (offcanvas click OR sessionId-bearing URL), client compares `session.question_json.problem_statement` (normalized whitespace) against `QUESTION_BY_ID[question_id].problem_statement`. If different, set `AppState.circlesStale = true`.
- All UI render paths read question content from `AppState.circlesSelectedQuestion` (which = `session.question_json` snapshot). NEVER re-fetch from current DB by id for display purposes.
- When `circlesStale === true`: render `.stale-banner` (banner above content) + force phase 1 read-only + phase 2 chat read-only + bottom action bar simplified to「回首頁」 only.
- Backend: server-side AI calls (gate / evaluate / chat / hint / coach / final-report) already pull `question_json` from session record — preserve this contract with a jest test that mocks a session whose `question_json.problem_statement` differs from current DB and asserts the LLM prompt uses the snapshot.
- New `scripts/audit-stale-sessions.js` — read-only DB sweep listing session ids whose `question_json` ≠ current DB content (for ops observability, no data mutation).

## 7 · Risks & mitigations

| Risk | Mitigation |
|---|---|
| Removing `.navbar { margin: 0 -16px }` breaks SP1's edge-to-edge already-shipped behavior | Test specifically checks navbar `getBoundingClientRect().left === 0` on every viewport |
| `q-card` flush at mobile/tablet but rounded at desktop creates implementation complexity | Use `@media (min-width: 1024px)` override block, document explicitly |
| pinnedCard removal breaks downstream binding | Grep audits already done (4 line refs); bind tests after removal |
| Backfill API cost / time for 100 questions | gpt-4o-mini is ~$0.0001/call ≈ $1 total; idempotent so re-runs safe |
| Q3 snapshot guard introduces inconsistency where prod LLM call + UI display use different sources | Single source of truth: server-side ALL pulls from `session.question_json`, frontend ALL reads from `AppState.circlesSelectedQuestion`. Jest test enforces. |
| Stale detection false positive on whitespace-only diff | Normalize whitespace (collapse runs, trim) before compare |
| Backwards compat: existing user with in-flight session encounters stale banner unexpectedly | Banner copy is non-blocking; user can dismiss by 「回首頁」; data not deleted |
