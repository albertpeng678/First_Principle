## 4. Phase 2 — 對話練習

### 4.1 頂部釘選題目卡
Phase 2 頂部固定顯示當前題目摘要卡片：
- 公司名 + 題型 badge
- 題目標題（1 行截斷，點可展開）
- 不隨對話捲動而消失

樣式：白底卡片，`border-bottom: 1px solid #e8e5de`，固定在 nav bar 下方。

### 4.2 每輪對話結構（已實作）
每輪 AI 回應分三區塊：
1. 被訪談者答覆（主要回應）
2. 教練點評（綠色 badge 標注）
3. 下一步提示（hint）

維持現有實作，不改動。

### 4.3 被訪談者 Prompt 修正
**問題**：使用者若直接問「公司的核心業務指標是什麼」，被訪談者直接給出答案，失去訓練意義。

**修正**：在 `prompts/circles-phase2.js`（或對應 prompt 檔案）的被訪談者角色指令中加入：

> 若使用者的提問過於直接、跳過應有的 PM 分析步驟（例如直接要求你說出核心指標、量化影響、或策略結論），請給予模糊、真實的利害關係人回應，例如：「這個我們內部有一些數據，但你覺得應該怎麼看這個問題？」或「我需要看看具體情況才能說。」不要直接給出答案。

### 4.4 提交步驟入口
在 input bar **下方**獨立一行：

```
[對話足夠了，提交這個步驟]  ← 文字按鈕，置中或靠右
```

- `font-size: 11px`，`color: #5a5a5a`
- `border: 1px solid #e8e5de`，`border-radius: 8px`
- 前 2 輪對話不顯示，第 3 輪起才出現
- 點擊觸發「結論提交流程」（見 4.5）

### 4.5 結論提交流程（三狀態）

**狀態 A：可收起條（預設）**

點擊「提交步驟」後，input bar 整列替換成一條收起條：

```
[整理結論]  翻閱完對話後，點右側展開填寫     [展開填寫 ▲]
```

- 對話記錄完整可捲動
- 讓使用者先回顧對話找線索再填寫

**狀態 B：展開填寫**

點「展開填寫」後，底部換成結論框：

- 標題：「整理你這個步驟確認了什麼」
- 可收起範例區塊（預設收起，顯示不同題目的範例結論，非當前題目）
- `textarea`（`height: 60px`，`resize: none`）
- 停止輸入 **8 秒**後觸發 AI 偵測（非關鍵字比對，呼叫 `/conclusion-check` 或複用現有 endpoint）
- AI 偵測結果：
  - 有缺漏 → 顯示黃底提示（「尚未提到…」）
  - 通過 → 顯示綠底確認，「確認提交」按鈕啟用
- 左下角「← 繼續對話」：點擊返回狀態 A 收起條（不清除已輸入內容）
- 右下角「確認提交」：預設灰色不可點，AI 偵測通過後變藍可點

**狀態 C：確認提交後**

進入 Phase 3 步驟評分（現有流程）。

## 6. 完整模擬總結報告（新頁面）

只有「完整模擬」模式走完全部 7 步驟後才顯示。路由：`view: 'circles-report'`。

### 6.1 頁面結構

**Header**
- 標題：「CIRCLES 完整模擬 — 總結」
- 右上角：「回首頁」文字連結

**題目資訊**
- 公司名 · 題目標題（`font-size: 11px`，`color: #5a5a5a`）

**總分 + AI 教練總評**
並排顯示：
- 左：總分圓環（`width: 64px`，`border: 3px solid #1A56DB`，數字 `font-size: 20px`，藍色）
- 右：白色卡片，標題「AI 教練總評」，1-2 句 AI 生成總評文字

**步驟分數 Pills**
7 個 pill，每個顯示：
- 步驟英文字母（藍色，`font-size: 9px`）
- 分數（`font-size: 12px`，`font-weight: 600`，`color: #1a1a1a`）
- 中文步驟名（`font-size: 8px`，`color: #5a5a5a`）
- 低分步驟（< 70 分）右上角加小藍點（`width: 5px`，`height: 5px`，`opacity: .5`）

提示文字：「點任一步驟查看詳細回饋」

**CTA 按鈕**
- 主要：「換題再練」（藍底白字）
- 次要：「針對弱項加練」（白底灰字）

### 6.2 步驟回溯機制

點擊任一 pill → 展開該步驟詳情卡（inline，pill 下方），其他 pill `opacity: 0.4`。

展開卡內容：
- **Header**：藍色背景，步驟字母圓形 badge + 步驟名稱 + 分數，右側「收起 ▲」
- **你的步驟結論**：顯示使用者在 Phase 2 提交的結論文字（`background: #f0ede6`）
- **教練點評**：
  - 「做得好：」（`font-weight: 700`）+ 說明文字（綠底框）
  - 「待加強：」（`font-weight: 700`）+ 說明文字 + 具體建議（黃底框）
- 再點同一 pill 或「收起」→ 關閉展開

### 6.3 AI 生成內容

總結報告需要以下 AI 呼叫（在最後一步驟評分完成時觸發，或進入報告頁時觸發）：
- **總評文字**：根據 7 個步驟的評分 + 使用者結論，生成 1-2 句整體評語
- 各步驟的教練點評已在 Phase 3 評分時生成，直接複用
