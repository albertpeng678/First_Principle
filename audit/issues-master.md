# Audit Issue Master List — 2026-04-30

## Severity rules used
- **P0** = blocks core flow / data loss / a11y violation / desktop layout broken at >=1440 / console error / cannot complete
- **P1** = major UX friction (broken sticky/modal, mobile tap target <44px, contrast fail, broken resume, copy that misled >=2 personas)
- **P2** = polish only
- **P3-runner-artefact** = surfaced by a limitation of `tests/playwright/journeys/audit/journey-runner.spec.js`, not the product itself

> Runner-artefact note: `journey-runner.spec.js` only fills the FIRST `textarea` per step (`fillFirstTextarea`), but CIRCLES Step C requires >=2 sub-fields filled before "送出評分" advances. Therefore the observation "screenshots 03–10 all show step C 1/7 with `請至少填寫 2 個欄位再送出評分`" is a runner artefact, not a product flow blocker. However, every layout / sticky-bar / copy / tap-target finding visible on those same screenshots IS a real product issue and is preserved as its own row below.
